🐝 Installation guide

Install Quizbee on your computer

No technical experience needed. Follow these steps and you'll have Quizbee running in about 10 minutes.

Step 1 — Install the prerequisites

Quizbee runs inside Docker, a tool that packages everything the app needs into self-contained containers. You only need to install Docker — it brings all other dependencies with it.

A

Install Docker Desktop for Windows

Go to docker.com/products/docker-desktop and click Download for Windows.

Run the installer (Docker Desktop Installer.exe). Accept the default settings. When prompted to enable WSL 2, click OK — Docker needs this to run on Windows.

After installation, restart your computer if asked.

B

Start Docker Desktop

Open the Docker Desktop app from your Start menu. Wait for the whale icon in your taskbar to stop animating — that means Docker is ready.

You don't need to sign in to a Docker account. Just close the "Sign in" prompt.

C

Open a terminal (Command Prompt or PowerShell)

Press Windows + R, type cmd, and press Enter. Or search for "PowerShell" in the Start menu. You'll type a few commands here in the next steps.

ℹ️ Windows 10 or 11? Both work. Docker Desktop requires Windows 10 version 2004 or later, or Windows 11 (any version).
A

Install Docker Desktop for Mac

Go to docker.com/products/docker-desktop and click Download for Mac.

If you're not sure which version, choose Mac with Apple chip for newer Macs (M1/M2/M3/M4) or Mac with Intel chip for older ones. You can check under Apple menu → About This Mac — look for "Apple M" (Apple chip) or "Intel" (Intel chip).

Open the downloaded .dmg file, drag Docker to Applications, and launch it.

B

Start Docker Desktop

Open Docker Desktop from your Applications folder. Wait for the whale icon in the menu bar to stop animating — Docker is ready.

You don't need to sign in to a Docker account. Just close any sign-in prompts.

C

Open Terminal

Press Command + Space, type Terminal, and press Enter. This is where you'll type a few commands in the next steps.

A

Install Docker Engine and Docker Compose

Open a terminal and run the official one-line installer:

curl -fsSL https://get.docker.com | sh

Then add your user to the docker group so you can run Docker without sudo: 

sudo usermod -aG docker $USER

Log out and log back in for the group change to take effect.

B

Verify Docker is running

docker --version
docker compose version

Both commands should print version numbers with no errors.

ℹ️ These instructions work on Ubuntu, Debian, Fedora, and most other common distributions. For enterprise Linux (RHEL, CentOS), see the Docker Engine install docs.

Step 2 — Download Quizbee

You need to download the Quizbee files to your computer. There are two ways: with Git (recommended) or by downloading a ZIP.

1

Option A — Download with Git (recommended)

In your terminal, navigate to the folder where you want to install Quizbee. For example, to put it in your Documents folder: 

# Windows (Command Prompt)
cd %USERPROFILE%\Documents

# Mac / Linux
cd ~/Documents

Then download Quizbee:

git clone https://github.com/mnavas/quizbee.git
cd quizbee
ℹ️ Don't have Git? On Windows you can install it from git-scm.com. On Mac it's usually already installed — if not, run git --version and follow the prompt. On Linux: sudo apt install git (Ubuntu/Debian) or sudo dnf install git (Fedora).

Using Git also makes it easy to update Quizbee later — see the Quick Reference at the bottom of this page.
2

Option B — Download as ZIP

Go to github.com/mnavas/quizbee, click the green Code button, then click Download ZIP.

Extract the ZIP to your Documents folder (or wherever you prefer). You'll have a folder called quizbee-main — open a terminal inside that folder.

Step 3 — Set your passwords

Quizbee needs a configuration file with your passwords and a security key. This takes about two minutes.

1

Copy the configuration template

In your terminal (inside the quizbee folder), run:

# Windows
copy .env.example .env

# Mac / Linux
cp .env.example .env

This creates a file called .env in the quizbee folder.

2

Edit the .env file

Open the .env file with any text editor (Notepad on Windows, TextEdit on Mac). You'll see something like this: 

DB_PASSWORD=change_this_strong_password
SECRET_KEY=change_this_64_char_random_string
ADMIN_EMAIL=admin@yourdomain.com
ADMIN_PASSWORD=change_this_password

Replace each value with your own:

• DB_PASSWORD — any strong password for the database (e.g. MyDb!Pass2026)
• SECRET_KEY — a long random string. Generate one by running: 

# Windows (PowerShell)
[Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Maximum 256 }) -as [byte[]])

# Mac / Linux
openssl rand -hex 32

Copy the output and paste it as the SECRET_KEY value.

• ADMIN_EMAIL — the email you'll use to log in as administrator
• ADMIN_PASSWORD — your administrator password

3

Save and close the file

Save the .env file. Make sure it's still named exactly .env (not .env.txt). On Windows, Notepad sometimes adds .txt — if that happens, rename the file in File Explorer (enable "Show file extensions" in View settings).

🔒 Keep your .env file private. Never share it or commit it to Git — it contains your passwords. It's already listed in .gitignore so Git will ignore it automatically.

Step 4 — Start Quizbee

One command builds and starts everything — the database, the API server, and the web interface. The first run takes a few minutes because Docker downloads the required software.

1

Start the server

In your terminal, inside the quizbee folder, run:

docker compose up -d

The -d flag runs Quizbee in the background so it keeps running even after you close the terminal.

The first time, Docker downloads all components — this can take 3–10 minutes depending on your internet speed. Subsequent starts are instant.

2

Open Quizbee in your browser

Once the command finishes, open your browser and go to: 

http://localhost:3000

You should see the Quizbee login page. Log in with the ADMIN_EMAIL and ADMIN_PASSWORD you set in Step 3.

3

Access from other devices on your network (optional)

To let students on the same Wi-Fi network take tests on their phones or tablets, find your computer's IP address: 

# Windows
ipconfig

# Mac / Linux
hostname -I

Look for an address starting with 192.168. or 10. (for example 192.168.1.42). Students open http://192.168.1.42:3000 in their browser.

The admin panel is the same URL — log in from any device on the network.

✅ Quizbee is running! Head to Examples to learn how to create your first test. You can also explore the Tests, Users, and Settings menus in the admin panel.

Quick reference — daily commands

All commands are run from the terminal inside the quizbee folder.

Server commands

Start server docker compose up -d
Stop server docker compose down
Restart server docker compose restart
View live logs docker compose logs -f
Check status docker compose ps

Update to the latest version

1. Pull updates git pull origin main
2. Rebuild docker compose build
3. Restart docker compose up -d

Your data is stored in Docker volumes and is not affected by updates. Always run all three commands in order when updating.

💡 Want Quizbee to start automatically when your computer boots? The restart: unless-stopped setting in the configuration means Docker will restart Quizbee automatically after a reboot — as long as Docker Desktop itself starts at login. On Windows and Mac, enable "Start Docker Desktop when you log in" in Docker Desktop Settings → General.
